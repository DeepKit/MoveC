一、准备工作
═══════════════════════════════════════════════════════════
[ ] 1. 运行 "创建测试数据.bat" 或 "Test-SyncFunction.ps1"
[ ] 2. 确认 D:\Test\Source 目录已创建
[ ] 3. 确认 D:\Test\Target 目录已创建
[ ] 4. 源目录包含测试文件（至少50个文件）

═══════════════════════════════════════════════════════════
二、手动同步功能测试 (约5分钟)
═══════════════════════════════════════════════════════════
[ ] 1. 创建手动同步任务
      - 任务名称：手动同步测试
      - 源路径：D:\Test\Source
      - 目标路径：D:\Test\Target
      - 同步模式：手动

[ ] 2. 执行首次同步
      [ ] 点击"开始同步"按钮
      [ ] 进度条正常显示
      [ ] 状态栏显示"正在同步"
      [ ] 完成后显示"同步完成"

[ ] 3. 验证同步结果
      [ ] 目标目录文件数与源目录一致
      [ ] 随机抽查5个文件，内容完全相同
      [ ] 子目录结构完全相同
      [ ] 文件大小匹配

[ ] 4. 增量同步测试
      [ ] 修改 Source\test1.txt
      [ ] 添加新文件 Source\test_new.txt
      [ ] 删除 Source\test2.txt
      [ ] 再次执行同步
      [ ] 验证 Target\test1.txt 已更新
      [ ] 验证 Target\test_new.txt 已创建
      [ ] 验证 Target\test2.txt 仍存在（手动同步不删除）

═══════════════════════════════════════════════════════════
三、实时同步功能测试 (约10分钟)
═══════════════════════════════════════════════════════════
[ ] 1. 清空目标目录 D:\Test\Target
      
[ ] 2. 创建实时同步任务
      - 任务名称：实时同步测试
      - 源路径：D:\Test\Source
      - 目标路径：D:\Test\Target
      - 同步模式：实时
      - 防抖间隔：500ms
      - 启用：是

[ ] 3. 等待初始同步（约10秒）
      [ ] 目标目录自动出现文件

[ ] 4. 测试实时添加
      [ ] 在 Source 中创建 realtime_test_1.txt
      [ ] 等待 2 秒
      [ ] 验证 Target 中出现该文件

[ ] 5. 测试实时修改
      [ ] 修改 Source\test1.txt 内容
      [ ] 等待 2 秒
      [ ] 验证 Target\test1.txt 内容已更新

[ ] 6. 测试实时删除
      [ ] 删除 Source\test2.txt
      [ ] 等待 2 秒
      [ ] 验证 Target\test2.txt 已删除

[ ] 7. 测试批量操作
      [ ] 快速添加5个文件到 Source
      [ ] 等待 3 秒
      [ ] 验证所有文件都已同步

[ ] 8. 查看日志
      [ ] 日志窗口显示实时同步事件
      [ ] 显示文件路径和操作类型

      [ ] .git 目录未同步

      [ ] 其他正常文件已同步
