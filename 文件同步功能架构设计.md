## 📐 架构设计概览

### 功能定位
文件同步功能作为 MoveC 的核心增值功能，为用户提供本地和局域网的文件自动同步能力，特别针对开发者（Delphi/Python/Node.js）的代码同步需求。

### 设计原则
1. **解耦设计**: 同步引擎独立于主程序，可单独测试和维护
2. **数据驱动**: 所有配置和状态持久化到 SQLite3 数据库
3. **性能优先**: 实时监控采用文件系统事件（非轮询），最小化资源占用
4. **容错机制**: 网络中断、磁盘满等异常场景自动重试和降级
5. **用户友好**: 预设模板降低配置门槛，状态可视化清晰直观

## 🗄️ 数据库设计

### 表结构设计（SQLite3）

#### 1. sync_tasks 表（同步任务）
```sql
CREATE TABLE sync_tasks (
    task_id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_name TEXT NOT NULL,                  -- 任务名称（用户自定义）
    source_path TEXT NOT NULL,                -- 源目录路径（本地或网络）
    target_path TEXT NOT NULL,                -- 目标目录路径
    sync_mode TEXT NOT NULL DEFAULT 'manual', -- 同步模式：'realtime'/'manual'
    is_enabled INTEGER DEFAULT 1,             -- 是否启用（0/1）
    preset_id INTEGER,                        -- 关联的预设ID（可为NULL）
    filter_rules TEXT,                        -- 过滤规则（JSON格式）
    last_sync_time DATETIME,                  -- 最后同步时间
    last_sync_status TEXT,                    -- 最后同步状态：'success'/'failed'/'running'/'paused'
    last_error_message TEXT,                  -- 最后错误信息
    files_synced INTEGER DEFAULT 0,           -- 已同步文件数
    bytes_synced INTEGER DEFAULT 0,           -- 已同步字节数
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (preset_id) REFERENCES sync_presets(preset_id)
);
```

#### 3. sync_history 表（同步历史记录）
```sql
CREATE TABLE sync_history (
    history_id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL,
    sync_start_time DATETIME,
    sync_end_time DATETIME,
    status TEXT NOT NULL,                     -- 'success'/'failed'/'cancelled'
    files_added INTEGER DEFAULT 0,
    files_modified INTEGER DEFAULT 0,
    files_deleted INTEGER DEFAULT 0,
    bytes_transferred INTEGER DEFAULT 0,
    error_message TEXT,
    FOREIGN KEY (task_id) REFERENCES sync_tasks(task_id) ON DELETE CASCADE
);

CREATE INDEX idx_sync_history_task ON sync_history(task_id);
CREATE INDEX idx_sync_history_time ON sync_history(sync_start_time DESC);
```

#### 4. sync_file_states 表（文件状态跟踪）
```sql
CREATE TABLE sync_file_states (
    state_id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL,
    file_relative_path TEXT NOT NULL,         -- 相对路径
    file_hash TEXT,                           -- SHA-256 哈希
    file_size INTEGER,
    last_modified DATETIME,
    sync_status TEXT,                         -- 'synced'/'pending'/'failed'
    last_sync_time DATETIME,
    FOREIGN KEY (task_id) REFERENCES sync_tasks(task_id) ON DELETE CASCADE,
    UNIQUE(task_id, file_relative_path)
);

CREATE INDEX idx_file_states_task ON sync_file_states(task_id);
```

## 🔧 技术实现要点

### 1. 文件系统监控
**Windows API**: ReadDirectoryChangesW
```pascal
function TFileSystemWatcher.Execute: Boolean;
var
  Buffer: array[0..8191] of Byte;
  BytesReturned: DWORD;
  NotifyInfo: PFileNotifyInformation;
begin
  FHandle := CreateFile(
    PChar(FWatchPath),
    FILE_LIST_DIRECTORY,
    FILE_SHARE_READ or FILE_SHARE_WRITE or FILE_SHARE_DELETE,
    nil,
    OPEN_EXISTING,
    FILE_FLAG_BACKUP_SEMANTICS or FILE_FLAG_OVERLAPPED,
    0
  );
  
  while not Terminated do
  begin
    if ReadDirectoryChangesW(FHandle, @Buffer, SizeOf(Buffer),
      True, FILE_NOTIFY_CHANGE_FILE_NAME or FILE_NOTIFY_CHANGE_SIZE or 
      FILE_NOTIFY_CHANGE_LAST_WRITE, @BytesReturned, nil, nil) then
    begin
      ProcessChanges(@Buffer);
    end;
  end;
end;
```

### 2. 局域网路径访问
```pascal
function ValidateNetworkPath(const APath: string): Boolean;
begin
  // 检查路径是否可访问
  Result := (APath.StartsWith('\\')) and DirectoryExists(APath);
  
  // 如果失败，尝试重新建立连接
  if not Result then
    Result := TryConnectNetworkDrive(APath);
end;
```

### 3. 增量同步算法
```pascal
procedure TSyncTask.IncrementalSync;
var
  SourceFiles, TargetFiles: TStringList;
  I: Integer;
  FileInfo: TSyncFileInfo;
begin
  // 1. 扫描源目录
  SourceFiles := ScanDirectory(FSourcePath, FFilterRules);
  
  // 2. 加载目标状态（从数据库）
  TargetFiles := LoadSyncedFileStates(FTaskID);
  
  // 3. 比对差异
  for I := 0 to SourceFiles.Count - 1 do
  begin
    FileInfo := SourceFiles.Objects[I] as TSyncFileInfo;
    
    if not TargetFiles.Contains(FileInfo.RelativePath) then
      QueueFileAction(faAdded, FileInfo)  // 新增文件
    else if IsFileModified(FileInfo) then
      QueueFileAction(faModified, FileInfo);  // 修改文件
  end;
  
  // 4. 检测删除（目标有但源没有）
  for I := 0 to TargetFiles.Count - 1 do
  begin
    if not SourceFiles.Contains(TargetFiles[I]) then
      QueueFileAction(faDeleted, TTargetFiles.Objects[I] as TSyncFileInfo);
  end;
  
  // 5. 执行同步操作
  ExecuteQueuedActions;
end;
```

### 4. 冲突处理策略
```pascal
type
  TConflictResolution = (crSourceWins, crTargetWins, crNewerWins, crAskUser);
  
procedure TSyncTask.HandleConflict(const AFile: TSyncFileInfo);
begin
  case FConflictResolution of
    crSourceWins: CopyFileEx(SourcePath, TargetPath, ...);
    crNewerWins: 
      if AFile.SourceModified > AFile.TargetModified then
        CopyFileEx(...)
      else
        // 保留目标文件
    crAskUser:
      ShowConflictDialog(AFile);
  end;
end;
```

## 📊 性能优化策略

### 1. 批量操作
- 文件变更事件缓冲 500ms，批量处理
- 数据库批量插入/更新（事务）

### 2. 智能哈希
- 小文件（< 1MB）：全量 SHA-256
- 中文件（1-100MB）：抽样哈希（头尾各 64KB）
- 大文件（> 100MB）：分块哈希 + 修改时间

### 3. 网络优化
- 局域网：直接复制
- 支持断点续传（分块传输）
- 连接池复用

### 4. 资源控制
- 最多 3 个并发同步任务
- 单任务最多 2 个并发文件传输
- 自动降级（CPU/磁盘负载过高时暂停）

## 🛡️ 容错机制

### 1. 异常处理
```pascal
try
  ExecuteSync;
except
  on E: ENetworkError do
    ScheduleRetry(60);  // 60秒后重试
  on E: EDiskFullError do
  begin
    PauseTask;
    NotifyUser('目标磁盘空间不足');
  end;
  on E: Exception do
  begin
    LogError(E);
    UpdateTaskStatus(ssFailed, E.Message);
  end;
end;
```

### 2. 重试策略
- 网络错误：指数退避（1分钟 → 5分钟 → 15分钟）
- 文件占用：重试 3 次，每次间隔 5 秒
- 权限错误：不重试，记录日志并跳过

### 3. 日志记录
- 所有同步操作记录到 `sync_history` 表
- 错误详情记录到 `sync_tasks.last_error_message`
- 关键事件输出到统一日志系统
