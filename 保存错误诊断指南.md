# 同步任务保存错误诊断指南

## 🔍 错误症状

创建任务时出现保存错误，需要逐步诊断问题。

---

## 📊 诊断步骤

### 步骤1：查看程序日志

运行程序并创建任务，查看日志窗口的详细输出：

```
✅ 正常流程应该显示：
[hh:mm:ss] 正在创建任务...
[hh:mm:ss] 任务对象创建成功: 测试持久化
[hh:mm:ss] 任务已添加到引擎
[hh:mm:ss] 任务保存完成 (ID: 1)
[hh:mm:ss] ✅ 新建任务成功: 测试持久化 (ID: 1)

❌ 如果看到错误：
[hh:mm:ss] ❌ 异常: [具体错误信息]
```

---

### 步骤2：使用DebugView查看详细调试信息

1. 下载并运行 DebugView (Sysinternals)
2. 以管理员身份运行
3. 运行程序并创建任务
4. 在DebugView中查看输出

**正常输出：**
```
SaveToDatabase: Preparing task data: 测试持久化
SaveToDatabase: Task data prepared, TaskID: 0
SaveToDatabase: Creating new task: 测试持久化
CreateSyncTask: Preparing SQL statement
CreateSyncTask: Setting parameters for task: 测试持久化
CreateSyncTask: Executing SQL
CreateSyncTask: Getting generated ID
CreateSyncTask: Generated ID: 1
SaveToDatabase: Task created with ID 1
```

**错误输出示例：**
```
SaveToDatabase: Exception: [具体错误信息]
CreateSyncTask: Exception: [具体错误信息]
```

---

### 步骤3：检查数据库文件

```powershell
# 检查数据库文件是否存在
Get-Item "MoveC.db" -ErrorAction SilentlyContinue

# 检查文件权限
Get-Acl "MoveC.db" | Format-List

# 查看文件大小
dir MoveC.db
```

---

## 🐛 常见错误及解决方法

### 错误1：数据库连接失败

**症状：**
```
[错误] 数据库连接失败: xxx\MoveC.db
```

**DebugView显示：**
```
CreateSyncTask: 数据库连接未建立
CreateSyncTask: 数据库连接已断开
```

**解决方法：**
1. 以管理员身份运行程序
2. 检查磁盘空间
3. 检查杀毒软件是否拦截
4. 删除MoveC.db让程序重新创建

---

### 错误2：SQL执行失败

**症状：**
```
❌ 异常: [SQLite错误信息]
```

**DebugView显示：**
```
CreateSyncTask: Exception: no such table: sync_tasks
CreateSyncTask: Exception: database is locked
```

**解决方法：**
1. 删除MoveC.db重新初始化
2. 确保没有其他程序占用数据库
3. 检查表结构是否正确

---

### 错误3：参数错误

**症状：**
```
❌ 异常: 参数类型不匹配
```

**DebugView显示：**
```
CreateSyncTask: Exception: datatype mismatch
```

**解决方法：**
1. 检查任务数据是否正确
2. 确保枚举值转换正确
3. 检查字符串长度是否超限

---

### 错误4：权限错误

**症状：**
```
❌ 异常: 无法写入文件
```

**DebugView显示：**
```
CreateSyncTask: Exception: unable to open database file
```

**解决方法：**
1. 以管理员身份运行
2. 将程序移到非系统目录
3. 添加杀毒软件白名单
