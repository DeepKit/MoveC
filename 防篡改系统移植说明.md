# 防篡改系统移植说明

本说明用于指导将现有“防篡改系统”集成到其它程序（Delphi/Windows 为例）。如与实际模块/工具名称不一致，请用项目中的实际名称替换文中的占位符（如 AntiTamper.Core、AntiTamper.CLI 等）。

- 目标：在构建与运行时检测程序本体与关键资源被修改/注入/挂钩，并采取响应措施。
- 适用范围：Win32/Win64 可执行程序或 DLL，VCL/FMX、服务程序、控制台程序。
- 术语：
  - 预构建/封印：编译后对二进制计算指纹并写入资源/节区以便运行时校验。
  - 运行时巡检：程序运行过程中周期性或事件触发的完整性检查。

---

## 1. 体系结构概览

模块构成（示例命名）：
- AntiTamper.Core：运行时核心库（单元/静态库），提供初始化、校验与策略。
- AntiTamper.Runtime.Win：平台相关实现（调试器检测、IAT/Inline Hook 检测等）。
- AntiTamper.Policy（JSON/YAML）：策略配置文件，或编译期生成的常量/资源。
- AntiTamper.CLI：命令行工具，用于预构建封印（计算哈希、签名并写入二进制）。

工作流程：
1) 构建阶段：
   - 收集目标二进制及（可选）依赖清单
   - 计算校验（.text/.rdata/导入表/自定义节区等）
   - 使用私钥签名指纹（可选）
   - 将指纹/公钥/策略写入资源或自定义节区（Seal/Stamp）
2) 运行阶段：
   - 早期初始化（尽量在入口点之前或 Application.Initialize 之前）
   - 自检主模块与关键区域，启动反调试与巡检任务
   - 发现异常时按策略响应（退出、限制、报警、延迟、诱导等）

## 2. 快速集成步骤

1) 引入文件/库
- 将 AntiTamper.Core、AntiTamper.Runtime.Win 等单元加入目标工程。
- 将 AntiTamper.CLI 与策略模板文件放入 `tools/antitamper`（建议路径）。

2) 在程序入口初始化（尽早）
- 对 GUI 程序：在 `.dpr` 中 `Application.Initialize` 之前调用。
- 对控制台/服务：在最早可执行点调用。

```pascal path=null start=null
program YourApp;

uses
  System.SysUtils,
  Vcl.Forms,
  AntiTamper.Core,
  AntiTamper.Runtime.Win,
  AntiTamper.Policy in 'config\antitamper.policy.pas',
  MainForm in 'src\MainForm.pas' {FMain};

{$R *.res}

begin
  // 尽早启动（可选启用严格模式与早期自检）
  AntiTamper.Bootstrap(DefaultPolicy, [atStrict, atEarlySelfCheck]);

  Application.Initialize;
  Application.MainFormOnTaskbar := True;
  Application.CreateForm(TFMain, FMain);
  Application.Run;
end.
```

3) 配置违规回调与策略（可选）
```pascal path=null start=null
uses AntiTamper.Types, AntiTamper.Actions;

procedure SetupTamperHandlers;
begin
  AntiTamper.OnViolation(
    procedure(const V: TViolation)
    begin
      // 记录并响应（示例：立刻退出并写日志）
      AntiTamper.Actions.Log(V);
      AntiTamper.Actions.Terminate(911); // 自定义退出码
    end
  );
end;
```

4) 添加预/后构建封印
- 在工程的 Pre-Build/Post-Build 事件中调用 CLI 工具完成指纹计算与写入。

示例（PowerShell，Post-Build）：
```powershell path=null start=null
$bin = "$(OUTPUTPATH)\$(PROJECTNAME).exe"  # 依据实际变量调整
$map = "$(OUTPUTPATH)\$(PROJECTNAME).map"  # 可选：定位节区/函数边界
$pol = "$(PROJECTDIR)\antitamper.policy.json"

& .\tools\antitamper\AntiTamper.CLI.exe seal `
  --binary "$bin" `
  --map "$map" `
  --policy "$pol" `
  --algo SHA256 `
  --embed resource `
  --public-key .\tools\antitamper\pubkey.pem
if ($LASTEXITCODE -ne 0) { throw "Anti-tamper sealing failed." }
```

## 3. 策略与配置

可通过 JSON 文件或编译期常量配置（示例）：
```json path=null start=null
{
  "hash": {
    "algorithm": "SHA256",
    "scope": [".text", ".rdata", "imports"],
    "excludeSections": [".rsrc"],
    "whitelistFiles": []
  },
  "runtime": {
    "earlySelfCheck": true,
    "periodicCheckMs": 5000,
    "randomJitterMs": 1500,
    "antiDebug": "strict",          
    "detectHooks": ["IAT", "INLINE"]
  },
  "response": {
    "onTamper": "terminate",        
    "onDebugger": "delay-exit",     
    "report": {
      "enabled": false,
      "endpoint": "",
      "timeoutMs": 1500
    }
  },
  "keys": {
    "publicKeyResource": "AT_PUBKEY"  
  }
}
```

关键配置说明：
- hash.scope：建议至少覆盖 `.text` 与导入表；如需更严，可加入 `.rdata`/自定义节。
- excludeSections：如版本信息、资源可能构建后被签名器重写，可排除或改为单独策略。
- antiDebug：`off|lenient|strict`。
- detectHooks：是否检测 IAT/Inline Hook、SSDT（驱动态不适用用户态）。
- response：违规响应策略；生产建议“限制/退出+可选上报”。

## 4. 运行时 API 参考（示例）

```pascal path=null start=null
// 初始化/引导
procedure AntiTamper.Bootstrap(const Policy: TAtPolicy; const Options: TAtOptions = []);

// 立即自检（返回是否通过）
function AntiTamper.VerifySelf: Boolean;

// 校验指定模块（DLL/EXE）
function AntiTamper.VerifyModule(const ModulePath: string): Boolean;

// 保护代码段（进入/退出期间启用更严的巡检/陷阱）
procedure AntiTamper.ProtectSectionBegin(const Name: string);
procedure AntiTamper.ProtectSectionEnd(const Name: string);

// 设置违规回调
procedure AntiTamper.OnViolation(const Handler: TViolationHandler);

// 动作库（日志/限流/退出/诱导延迟/断连等）
namespace AntiTamper.Actions
  procedure Log(const V: TViolation);
  procedure Terminate(ExitCode: Integer);
  procedure Slowdown(Ms: Integer);
  procedure SandboxMode(Enabled: Boolean);
end;
```

使用示例：
```pascal path=null start=null
AntiTamper.ProtectSectionBegin('LicenseCheck');
try
  PerformLicenseValidation;
finally
  AntiTamper.ProtectSectionEnd('LicenseCheck');
end;
```

## 5. 构建与发布建议

- 签名顺序：先“封印”(seal)再“代码签名”(Authenticode)。封印时排除后续会被签名器改动的区域，或在签名后再次更新策略中的签名指纹（取决于工具链）。
- 私钥管理：不入库；CI 用临时安全存储（如环境机密/安全库），构建时解封装到内存使用。
- 产物清单：主程序、策略（若不嵌入）、公钥/证书、CLI 日志（便于追溯）。
- CI 集成：在流水线的“构建后、签名前”增加封印步骤，并在冒烟测试阶段做一次自检运行。

## 6. 与 Delphi 工程集成要点

- 工程事件：RAD Studio/MSBuild 可在 Project Options 的 Build Events 配置 Pre/Post-Build。变量名可能为 `$(ProjectDir)`, `$(Config)`, `$(Platform)`, `$(OutputPath)`, `$(ProjectName)`，请按实际版本核对。
- MAP 文件：建议开启以利于更精确的节区/函数边界封印。
- 服务程序：在 `ServiceStart` 之前调用 `Bootstrap`，并避免长阻塞；回调中不要做交互。
- DLL：在 `DllMain` 或 `initialization` 节中轻量初始化，避免加载死锁；可延迟到首次导出函数调用时自检。
- 第三方壳/虚拟化：与加壳顺序需实测，常见为：编译 → 封印 → 加壳/虚拟化 → 代码签名；或将封印嵌入加壳前镜像区域。

## 7. 常见问题与排查

- 启动变慢：减少 scope 或关闭首次全量校验，改为节流/抽样校验。
- 杀软误报：开启代码签名；避免异常 API 频繁调用；与白名单厂商对接。
- 调试开发：提供 `DEV_MODE` 或 `antiDebug=off` 的开发配置，但严禁与生产混用。
- 崩溃/退出无日志：确保回调中先写日志再退出；日志目录具写权限；避免在异常过滤器里做重型操作。
- 兼容性：针对老系统关闭高级反调试特性（如硬件断点扫描）。

## 8. 验证清单（上线前务必完成）

功能校验：
- 修改 EXE 某字节或替换依赖 DLL，程序应检测并按策略响应。
- 运行调试器/注入常见 Hook（如 IAT hook kernel32!CreateFileW），应被检测。
- 在策略中调整为 `report.enabled=true`，验证上报超时与失败退避。

稳健性：
- 在无网络/低权限/只读目录下运行不崩溃；日志与响应可用。
- 反复启动 100 次无假阳性；CPU/内存增幅可接受。

发布流程：
- 构建日志包含 Seal 成功记录与指纹摘要。
- 代码签名有效；时间戳可验证。
- 产物哈希入库；运维值班手册更新了响应指引。

## 9. 附录

A) Post-Build 执行示例（批处理）：
```bat path=null start=null
set BIN=%OUTPUTPATH%\%PROJECTNAME%.exe
set POL=%PROJECTDIR%\antitamper.policy.json

"tools\antitamper\AntiTamper.CLI.exe" seal ^
  --binary "%BIN%" ^
  --policy "%POL%" ^
  --algo SHA256 ^
  --embed resource
if errorlevel 1 exit /b 1
```

B) 运行时自检开关（条件编译示例）：
```pascal path=null start=null
{$IFDEF PRODUCTION}
  AntiTamper.Bootstrap(DefaultPolicy, [atStrict, atEarlySelfCheck]);
{$ELSE}
  AntiTamper.Bootstrap(DefaultPolicy, []); // 开发环境放宽
{$ENDIF}
```
