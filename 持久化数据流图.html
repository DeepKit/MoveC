🔍 关键检查点

✅ 检查点1：数据库连接

位置：MainSyncForm.FormCreate()
检查：FDatabase.Connected = True
日志："[成功] 数据库已连接"
失败影响：所有持久化操作失败

✅ 检查点2：任务对象创建

位置：TaskEditForm.CreateTask()
检查：NewTask <> nil
验证：必填字段完整
失败影响：无法保存到数据库

✅ 检查点3：SQL执行

位置：TSyncDatabase.CreateSyncTask()
检查：FQuery.ExecSQL() 成功
验证：无SQL异常
失败影响：任务无法插入

✅ 检查点4：ID获取

位置：GetLastAutoGenValue()
检查：返回值 > 0
日志："Generated ID: 1"
失败影响：TaskID保持为0

✅ 检查点5：ID回写

位置：TSyncTask.SaveToDatabase()
检查：FTaskID := NewID 执行
验证：FTaskID > 0
失败影响：界面显示保存失败

⚠️ 常见失败点

❌ 失败点1：数据库连接失败

症状："[错误] 数据库连接失败"
原因：文件权限、路径错误、磁盘空间
解决：管理员运行、检查路径、清理空间

❌ 失败点2：SQL执行失败

症状："❌ 异常: [SQL错误]"
原因：表结构错误、参数类型不匹配
解决：重建数据库、检查参数绑定

❌ 失败点3：ID获取失败

症状："Generated ID: -1"
原因：自增字段设置问题
解决：检查表结构、重新初始化

🎯 快速问题定位指南

根据日志信息定位：

  • "[错误] 数据库连接失败" → 检查数据库文件和权限
  • "❌ 创建任务对象失败" → 检查表单验证
  • "❌ 异常: [SQL错误]" → 检查SQL执行
  • "TaskID = 0" → 检查ID回写逻辑
  • "已加载 0 个同步任务" → 检查数据库内容

根据测试步骤定位：

  • 步骤1失败 → 数据库连接问题
  • 步骤2成功 → 对象创建正常
  • 步骤3失败 → SQL执行问题
  • 步骤4失败 → ID回写问题
  • 步骤5失败 → 数据读取问题

💡 使用建议：

  1. 按照流程图逐步执行测试
  2. 在每个检查点验证预期值
  3. 使用DebugView查看详细调试信息
  4. 根据失败症状快速定位问题
  5. 参考数据流向追踪表检查关键变量
