### 2. 动态密钥机制

**密钥组成**:
```
最终密钥 = 基础密钥 + (程序名 + 修改时间 + 盐值)
```

**优势**:
- 绑定程序版本
- 难以提取
- 自动更新
- 增强安全性

## 🔧 技术实现

### 加密流程

```
原始图像数据
    ↓
PKCS7填充（16字节对齐）
    ↓
生成随机IV（16字节）
    ↓
SHA-256密钥派生
    ↓
AES-256-CBC加密
    ↓
IV + 加密数据 → 数据库
```

### 解密流程

```
数据库加密数据
    ↓
分离IV和加密数据
    ↓
SHA-256密钥派生
    ↓
AES-256-CBC解密
    ↓
移除PKCS7填充
    ↓
原始图像数据
```

### 代码示例

```pascal
// 加密（自动选择算法）
case FConfig.EncryptionType of
  etXOR:
    Result := SimpleXOREncrypt(ImageData, FConfig.EncryptionKey);
  etAES256:
    Result := TBasicProtection.EncryptBinaryData(ImageData, FConfig.EncryptionKey);
end;

// 动态密钥
class function TAntiTamperPackage.GetDynamicEncryptionKey: string;
begin
  BaseKey := FConfig.EncryptionKey;
  DynamicPart := TBasicProtection.GetDynamicKey;
  Result := BaseKey + DynamicPart;
end;
```

## ⚠️ 重要提醒

### 1. 必须重新导入图像 ⚠️

由于加密算法改变，现有数据库中的图像将无法解密。

**操作**:
```bash
dcc32 ImportImages.dpr
ImportImages.exe
```

### 2. 不兼容旧数据

- AES-256加密的数据无法用XOR解密
- XOR加密的数据无法用AES-256解密
- 需要选择一种加密方式并重新导入

### 3. 密钥管理

**不要修改**:
- 程序文件名（影响动态密钥）
- 基础密钥字符串（影响解密）
