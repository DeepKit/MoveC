## 🚀 快速开始测试（5分钟）

### 步骤 1：创建测试数据

**选择一种方式：**

**方式 A：使用批处理（简单）**
```cmd
双击运行：创建测试数据.bat
```

**方式 B：使用 PowerShell（推荐）**
```powershell
.\Test-SyncFunction.ps1
```

### 步骤 2：启动同步程序

运行程序：`C盘超级瘦身.exe`

### 步骤 3：创建测试任务

1. 点击"新建任务"
2. 填写：
   - 任务名称：`测试任务`
   - 源路径：`D:\Test\Source`
   - 目标路径：`D:\Test\Target`
   - 同步模式：先选"手动同步"
3. 点击"确定"

### 步骤 4：执行首次同步

1. 选中刚创建的任务
2. 点击"开始同步"
3. 观察进度条和日志输出

### 步骤 5：验证结果

打开文件管理器：
```
D:\Test\Target\  ← 应包含 Source 的所有文件
```

**快速验证：**
- 文件数量是否一致
- 随机打开几个文件，内容是否相同

## ⚠️ 测试注意事项

### 1. 测试前检查

- ✅ 确保有足够磁盘空间（至少 500MB）
- ✅ 关闭杀毒软件的实时保护（可能影响性能）
- ✅ 确保测试目录没有被其他程序占用

### 2. 测试期间注意

- ⚠️ 不要在源目录进行大量操作（可能触发监控限制）
- ⚠️ 实时同步测试时，操作后等待 1-2 秒观察结果
- ⚠️ 查看日志窗口了解详细执行情况

## 🐛 问题排查

### Q: 实时同步不工作？

**检查：**
1. 任务是否启用（勾选"启用"）
2. 源路径是否存在
3. 日志窗口是否有错误信息
4. 防抖间隔是否太长

### Q: 文件没有同步？

**检查：**
1. 是否匹配忽略规则
2. 文件修改时间是否早于目标文件
3. 文件是否被占用（如 Word 打开的文档）

### Q: 同步很慢？

**可能原因：**
1. 磁盘性能（机械硬盘 vs SSD）
2. 杀毒软件扫描
3. 文件数量过多（建议分批或使用忽略规则）
