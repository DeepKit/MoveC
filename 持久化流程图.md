## ⚠️ 关键检查点

### 🔍 检查点1：数据库连接
```
MainSyncForm.FormCreate()
  │
  ├─ FDatabase := TSyncDatabase.Create(DbPath)
  ├─ FDatabase.Connect()
  │   ├─ 成功 → 继续
  │   └─ 失败 → ❌ 所有持久化操作失败
  └─ 日志: "[成功] 数据库已连接" 或 "[错误] 数据库连接失败"
```

### 🔍 检查点2：任务对象创建
```
TaskEditForm.CreateTask()
  │
  ├─ 读取表单数据
  ├─ NewTask := TSyncTask.CreateWithDatabase(FDatabase)
  ├─ 设置任务属性
  │   ├─ Name, SourcePath, TargetPath
  │   ├─ Mode, ConflictStrategy
  │   └─ Enabled, FilterRules
  └─ 返回 NewTask
      ├─ 成功 → 有效对象
      └─ 失败 → nil (需要检查)
```

### 🔍 检查点3：数据库写入
```
TSyncDatabase.CreateSyncTask()
  │
  ├─ 检查连接状态
  ├─ 准备 SQL: INSERT INTO sync_tasks...
  ├─ 绑定参数
  │   ├─ :name, :source_path, :target_path
  │   ├─ :sync_mode, :conflict_strategy
  │   ├─ :is_enabled, :filter_rules
  │   └─ :preset_id
  ├─ FQuery.ExecSQL()
  ├─ GetLastAutoGenValue()
  │   ├─ 成功 → 返回新ID (1, 2, 3...)
  │   └─ 失败 → 返回 -1
  └─ 日志: "Generated ID: X"
```

### 🔍 检查点4：ID回写
```
TSyncTask.SaveToDatabase()
  │
  ├─ NewID := FDatabase.CreateSyncTask(DBTask)
  ├─ if NewID > 0 then
  │   ├─ FTaskID := NewID  ← 关键步骤！
  │   └─ 保存成功
  └─ else
      └─ 保存失败 (TaskID 保持为 0)
```

## 🐛 常见失败点

### ❌ 失败点1：数据库连接失败
```
位置: MainSyncForm.FormCreate()
症状: "[错误] 数据库连接失败"
影响: 所有持久化操作失败
检查: 
  - 数据库文件路径
  - 文件权限
  - 磁盘空间
```

### ❌ 失败点2：任务对象无效
```
位置: TaskEditForm.CreateTask()
症状: "创建任务对象失败"
影响: 无法保存到数据库
检查:
  - 表单验证
  - 必填字段
  - 对象创建
```

### ❌ 失败点3：SQL执行失败
```
位置: TSyncDatabase.CreateSyncTask()
症状: "Exception: [SQL错误]"
影响: 任务无法插入数据库
检查:
  - 表结构
  - 参数类型
  - SQL语法
```

### ❌ 失败点4：ID获取失败
```
位置: GetLastAutoGenValue()
症状: "Generated ID: -1"
影响: TaskID保持为0，显示保存失败
检查:
  - 自增字段设置
  - SQLite版本兼容性
```

### ❌ 失败点5：ID未回写
```
位置: TSyncTask.SaveToDatabase()
症状: "Task created with ID: 1" 但界面显示 ID: 0
影响: 用户看到保存失败
检查:
  - FTaskID := NewID 是否执行
  - 对象引用是否正确
```

## 🔧 调试数据流

### 数据流向追踪
```
用户输入 → 表单验证 → 任务对象 → 内存列表 → 数据库写入 → ID回写 → 界面更新
    ↓           ↓          ↓         ↓         ↓         ↓         ↓
  检查必填   检查格式   检查对象   检查列表   检查SQL   检查ID   检查显示
```

### 关键变量追踪
```pascal
// 1. 数据库连接
FDatabase.Connected  // 应该是 True
FDatabase.Database   // 应该指向 MoveC.db

// 2. 任务对象
NewTask <> nil       // 应该不为空
NewTask.TaskID = 0   // 新建时为0
NewTask.Name <> ''   // 应该有名称

// 3. 保存过程
NewID > 0            // 数据库返回的ID
FTaskID := NewID     // ID回写
FTaskID > 0          // 最终检查

// 4. 界面显示
TaskID in log        // 日志中的ID
Task in ListView     // 列表中的任务
```

## 🎯 快速问题定位

### 根据日志定位
```
如果看到: "[错误] 数据库连接失败"
  → 检查数据库文件和权限

如果看到: "❌ 创建任务对象失败"
  → 检查表单验证和对象创建

如果看到: "❌ 异常: [SQL错误]"
  → 检查SQL执行和表结构

如果看到: "TaskID = 0"
  → 检查ID回写逻辑

如果看到: "已加载 0 个同步任务"
  → 检查数据库是否有数据
```

### 根据步骤定位
```
步骤1失败 → 数据库连接问题
步骤2成功 → 对象创建正常
步骤3失败 → SQL执行问题
步骤4失败 → ID回写问题
步骤5失败 → 数据读取问题
```
