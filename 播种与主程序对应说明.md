# 播种工具与主程序对应关系说明

## 一、图像 Key 对应关系

### 播种工具 → 数据库 → 主程序

```
播种工具                    数据库 MoveC.db              主程序代码
─────────────────────────────────────────────────────────────────
文件名（不含扩展名）    →   image_key 字段      →   LoadSecureImage() 参数
─────────────────────────────────────────────────────────────────
wechat.png              →   'wechat'            →   'wechat'
alipay.png              →   'alipay'            →   'alipay'  
btc.png                 →   'btc'               →   'btc'
其他.png                →   '其他'              →   '其他'
```

### 关键约束

1. **图像 Key = 文件名（不含扩展名）**
   - 播种时自动使用：`TPath.GetFileNameWithoutExtension(文件路径)`
   - 例如：`wechat.png` → key 为 `'wechat'`

2. **主程序读取时必须使用相同的 Key**
   ```pascal
   // 主程序中的调用示例
   TAntiTamperPackage.LoadSecureImage(TableImages, 'wechat', imgWechat, AddressText);
   //                                              ^^^^^^^^
   //                                              必须与播种时的文件名一致
   ```

3. **大小写敏感**
   - `wechat` ≠ `Wechat` ≠ `WECHAT`
   - 建议统一使用小写

## 二、主程序密码配置位置

### 当前密码配置在 `uMain.pas` 中

```pascal
// 文件：uMain.pas
// 位置：FormCreate 过程中

procedure TfrmMain.FormCreate(Sender: TObject);
var
  Config: TAntiTamperConfig;
begin
  // ...
  
  // 初始化防篡改包
  Config := TAntiTamperPackage.GetDefaultConfig;
  Config.EncryptionKey := 'MoveC_AntiTamper_Key_2025';  // ← 主程序密码在这里
  Config.DownloadURL := 'https://www.goodmem.cn';
  Config.EnableLogging := {$IFDEF DEBUG}True{$ELSE}False{$ENDIF};
  Config.EncryptionType := etAES256;
  Config.Salt := 'MoveC_Salt_v1';
  Config.KdfIterations := 10000;
  Config.EnableHMAC := True;
  TAntiTamperPackage.Initialize(Config);
  
  // ...
end;
```

### 密码一致性要求

**播种工具输入的密码 = 主程序 Config.EncryptionKey**

| 项目 | 值 |
|------|------|
| **主程序密码** | `MoveC_AntiTamper_Key_2025` |
| **播种工具输入** | 必须输入完全相同的密码 |
| **Salt** | `MoveC_Salt_v1`（双方必须一致）|
| **KDF迭代次数** | `10000`（双方必须一致）|

### 为什么主程序不需要用户输入密码？

**设计思路：分离播种与使用**

1. **播种阶段**（管理员）
   - 使用 `SeedTool.exe`
   - 输入管理员密码
   - 加密图像并写入数据库
   - **播种工具不分发给用户**

2. **使用阶段**（最终用户）
   - 运行 `C盘超级瘦身.exe`
   - 程序内置密码（硬编码在 `uMain.pas`）
   - 自动解密和校验
   - **用户不需要知道密码**

### 安全机制

```
管理员（播种）                    最终用户（使用）
─────────────────────────────────────────────────────
SeedTool.exe                     C盘超级瘦身.exe
  ↓ 输入密码                       ↓ 内置密码
  ↓ 加密图像                       ↓ 读取数据库
  ↓ 写入 MoveC.db                  ↓ 解密图像
                                   ↓ SHA-256 校验
分发：                              ↓ HMAC 校验
  C盘超级瘦身.exe               →   ✓ 校验通过 → 显示
  MoveC.db                         ✗ 校验失败 → 安全退出
```

```
   [21:05:18] ✓ 播种成功: wechat
   [21:05:18] ✓ 播种成功: alipay
   [21:05:18] ✓ 播种成功: btc
```

### 步骤 3：主程序中使用图像

```pascal
// 在主程序中读取图像
// 文件：FrameAboutMe.pas 或其他窗体

procedure TFrameAboutMe.LoadImages;
var
  AddressText: string;
begin
  // 加载微信收款码
  if TAntiTamperPackage.LoadSecureImage(
       TableImages,    // 数据库表
       'wechat',       // ← 对应 wechat.png
       imgWechat,      // 图像控件
       AddressText     // 输出地址文本
     ) then
    lblWechatAddress.Caption := AddressText;
    
  // 加载支付宝收款码
  if TAntiTamperPackage.LoadSecureImage(
       TableImages,
       'alipay',       // ← 对应 alipay.png
       imgAlipay,
       AddressText
     ) then
    lblAlipayAddress.Caption := AddressText;
    
  // 加载比特币地址
  if TAntiTamperPackage.LoadSecureImage(
       TableImages,
       'btc',          // ← 对应 btc.png
       imgBTC,
       AddressText
     ) then
    lblBTCAddress.Caption := AddressText;
end;
```

## 四、常见问题

### Q1：修改主程序密码后，旧数据库还能用吗？

**不能。** 需要：
1. 在 `uMain.pas` 中修改 `Config.EncryptionKey`
2. 使用新密码重新运行 `SeedTool.exe` 播种
3. 生成新的 `MoveC.db`

### Q2：如何添加新的图像？

1. 准备新图像文件（如 `new_image.png`）
2. 运行 `SeedTool.exe`
3. 输入密码：`MoveC_AntiTamper_Key_2025`
4. 添加 `new_image.png`
5. 播种（会自动添加到现有数据库）
6. 在主程序中调用：
   ```pascal
   LoadSecureImage(TableImages, 'new_image', imgControl, AddressText)
   ```

### Q3：数据库被篡改会怎样？

主程序加载时会：
1. 读取密文
2. 使用内置密码解密
3. 验证 SHA-256（完整性）
4. 验证 HMAC-SHA256（防篡改）
5. **任何一步失败 → 弹出安全警告 → 程序退出**

### Q4：用户能修改密码吗？

**不能。** 密码硬编码在主程序中，用户看不到也无法修改。这是 **fail-closed** 设计：
- 数据库 + 主程序是配套的
- 只有管理员知道密码
- 任何篡改都会导致程序拒绝运行

## 五、文件分发清单

### 给最终用户

```
✓ C盘超级瘦身.exe      （主程序，内置密码）
✓ MoveC.db              （已加密的数据库）
✗ SeedTool.exe          （不分发，仅管理员使用）
```

### 管理员保留

```
✓ SeedTool.exe          （播种工具）
✓ 原始图像文件          （wechat.png, alipay.png, btc.png 等）
✓ 密码备份              （MoveC_AntiTamper_Key_2025）
```

## 六、安全建议

1. **密码强度**
   - 当前密码：`MoveC_AntiTamper_Key_2025`（27字符）
   - 建议：定期更换更强的密码

2. **播种工具安全**
   - 仅在安全环境运行
   - 播种完成后立即关闭
   - 不要分发给用户

3. **数据库保护**
   - 定期备份 `MoveC.db`
   - 验证文件完整性（可用 SHA-256）
   - 避免在不安全环境中打开

4. **版本管理**
   - 修改密码 = 新版本
   - 旧版数据库无法被新版主程序读取
   - 需要重新播种并重新分发
