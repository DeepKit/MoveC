# 数据库持久化测试

## 已修复的问题

### 1. 数据库未正确连接 ✅
**原因：**
- 之前代码 `FDatabase := TSyncDatabase.Create;` 缺少数据库路径参数
- 没有调用 `Connect()` 方法实际连接数据库

**修复：**
```pascal
// MainSyncForm.pas FormCreate
var
  DbPath: string;
begin
  // 获取数据库路径
  DbPath := TPath.Combine(ExtractFilePath(ParamStr(0)), 'MoveC.db');
  
  // 创建数据库对象并传入路径
  FDatabase := TSyncDatabase.Create(DbPath);
  
  // 连接数据库
  if not FDatabase.Connect then
  begin
    ShowMessage('数据库连接失败，程序可能无法正常工作');
  end;
```

### 2. TaskEditForm 类型错误 ✅
**原因：**
- `FTask` 被错误地声明为指针类型 `^TSyncTask`
- 导致访问字段时出错

**修复：**
```pascal
// TaskEditForm.pas
private
  FTask: TSyncTask;  // 改回对象类型
  FIsNewTask: Boolean;  // 新增标志
```

### 3. 验证逻辑错误 ✅
**原因：**
- `ValidateInput` 尝试访问未初始化的 `FTask` 字段

**修复：**
- 移除了对 FTask 字段的提前访问
- 简化了验证逻辑

## 故障排查

### 问题：程序启动时提示"数据库连接失败"

**可能原因：**
1. 数据库文件不存在
2. 文件权限不足
3. SQLite 驱动未安装

**解决方案：**
```pascal
// 检查 FDConnection 配置
FConnection.DriverName := 'SQLite';
FConnection.Params.Values['Database'] := DbPath;
FConnection.LoginPrompt := False;
```

### 问题：任务创建后关闭程序再打开消失了

**检查：**
1. ✅ 是否调用了 `NewTask.Save`
2. ✅ `FDatabase.Connect` 是否返回 True
3. ✅ 数据库文件是否有写入权限

### 问题：删除任务后重启仍然出现

**检查：**
1. ✅ 是否调用了 `FDatabase.DeleteSyncTask(TaskID)`
2. ✅ TaskID 是否正确（> 0）

## 关键代码位置

### 新建任务
```pascal
// MainSyncForm.pas - NewTask1Click
NewTask := EditForm.CreateTask;
FSyncEngine.AddTask(NewTask);
NewTask.Save;  // 保存到数据库 ← 关键
RefreshTaskList;
```

### 编辑任务
```pascal
// MainSyncForm.pas - EditTask1Click
EditForm.LoadTask(FSelectedTask);
if EditForm.ShowModal = mrOk then
begin
  FSelectedTask.Save;  // 保存修改 ← 关键
  RefreshTaskList;
end;
```

### 删除任务
```pascal
// MainSyncForm.pas - DeleteTask1Click
TaskID := FSelectedTask.TaskID;
FRealtimeManager.StopRealtimeSync(TaskID);
FSyncEngine.RemoveTask(FSelectedTask);

// 从数据库删除 ← 关键
if Assigned(FDatabase) and (TaskID > 0) then
  FDatabase.DeleteSyncTask(TaskID);
```
