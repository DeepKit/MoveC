## 🔐 AES-256加密详解

### 什么是AES-256？

**AES** (Advanced Encryption Standard) 是美国国家标准与技术研究院(NIST)在2001年确立的加密标准。

- **256位密钥** - 提供2^256种可能性
- **CBC模式** - 密码块链接模式，增强安全性
- **随机IV** - 每次加密使用不同的初始化向量
- **PKCS7填充** - 标准数据填充方式

### 为什么选择AES-256？

1. **军事级加密** - 美国政府用于保护机密信息
2. **无已知漏洞** - 至今未被破解
3. **行业标准** - 银行、政府、企业广泛使用
4. **性能优异** - 硬件加速支持

### XOR vs AES-256 对比

| 特性 | XOR | AES-256 |
|------|-----|---------|
| 密钥长度 | 任意 | 256位 |
| 破解难度 | 极易 | 几乎不可能 |
| 安全性 | ⭐ | ⭐⭐⭐⭐⭐ |
| 标准化 | 无 | NIST标准 |
| 使用场景 | 简单混淆 | 机密数据保护 |

### 3. 加密方法升级

```pascal
class function TAntiTamperPackage.EncryptImageData(const ImageData: TBytes): TBytes;
begin
  if not FInitialized then
    raise Exception.Create('防篡改包未初始化');
  
  // 根据配置选择加密算法
  case FConfig.EncryptionType of
    etXOR:
      Result := SimpleXOREncrypt(ImageData, FConfig.EncryptionKey);
    etAES256:
      Result := TBasicProtection.EncryptBinaryData(ImageData, FConfig.EncryptionKey);
  else
    raise Exception.Create('未知的加密类型');
  end;
end;
```

### 4. 动态密钥生成

```pascal
class function TAntiTamperPackage.GetDynamicEncryptionKey: string;
var
  BaseKey: string;
  DynamicPart: string;
begin
  BaseKey := FConfig.EncryptionKey;
  
  // 使用uBasicProtection的动态密钥生成
  // 结合：程序名称 + 最后修改时间 + 固定盐值
  DynamicPart := TBasicProtection.GetDynamicKey;
  
  // 组合基础密钥和动态部分
  Result := BaseKey + DynamicPart;
end;
```

### 5. AES加密流程

```
原始数据
    ↓
PKCS7填充（16字节对齐）
    ↓
生成随机IV（16字节）
    ↓
SHA-256密钥派生
    ↓
AES-256-CBC加密
    ↓
IV + 加密数据
```

## 🎯 动态密钥详解

### 密钥组成

```
最终密钥 = 基础密钥 + 动态部分

动态部分 = 程序名称 + 最后修改时间 + 固定盐值
```

### 示例

```
基础密钥: "MoveC_AntiTamper_Key_2025"
程序名称: "C盘瘦身"
修改时间: "1728576000" (Unix时间戳)
固定盐值: "SecureKey2024"

动态部分: "C盘瘦身1728576000SecureKey2024"
最终密钥: "MoveC_AntiTamper_Key_2025C盘瘦身1728576000SecureKey2024"
```

### 优势

1. **绑定程序** - 不同程序无法解密
2. **绑定版本** - 修改程序后密钥改变
3. **难以提取** - 密钥动态计算，不存储
4. **自动更新** - 程序更新后自动使用新密钥

## ⚠️ 重要注意事项

### 1. 向后兼容性

**不兼容！** AES-256加密的数据无法用XOR解密。

**解决方案**:
- 保留XOR支持（`etXOR`）
- 新数据使用AES-256
- 旧数据需重新导入

### 2. 密钥管理

**不要修改**:
- 程序文件名
- 基础密钥字符串

否则将无法解密现有数据！

### 3. 性能考虑

- 首次加载略慢（密钥派生）
- 后续操作正常
- 建议使用SSD硬盘
