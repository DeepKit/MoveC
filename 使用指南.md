# 中文编码修复工具使用指南

## 快速开始

如果您遇到中文字符在数据库中变成乱码的问题，请按以下步骤操作：

### 1. 运行修复工具
```bash
# 编译修复工具（如果需要）
dcc32 -U"Core" FixAllLanguageFiles.dpr

# 运行修复工具
.\FixAllLanguageFiles.exe
```

### 2. 验证修复结果
```bash
# 编译验证程序（如果需要）
dcc32 -U"Core" VerifyChineseSupport.dpr

# 运行验证程序
.\VerifyChineseSupport.exe
```

### 3. 检查结果
验证程序应该显示所有测试都通过（✓ 成功）。

## 工具说明

### FixAllLanguageFiles.exe
**功能**：修复所有语言文件的编码问题
- 重新生成正确的UTF-8编码语言文件
- 修复简体中文、繁体中文、日文等语言文件
- 测试数据库中文支持功能

**输出文件**：
- `Languages/zh-CN.json` - 简体中文
- `Languages/zh-TW.json` - 繁体中文
- `Languages/ja-JP.json` - 日文

### VerifyChineseSupport.exe
**功能**：全面验证中文编码支持
- 测试文件编码读取
- 测试数据库中文操作
- 测试语言管理器
- 测试完整工作流程

## 常见问题

### Q: 为什么终端显示乱码？
**A**: 这是正常现象。Windows终端默认使用GBK编码，而我们的文件使用UTF-8编码。程序运行时会正确显示中文。

### Q: 如何确认修复是否成功？
**A**: 运行 `VerifyChineseSupport.exe`，如果所有测试都显示"✓ 成功"，说明修复成功。

### Q: 修复后需要重新编译主程序吗？
**A**: 不需要。修复的是数据文件和语言文件，主程序会自动使用修复后的文件。

### Q: 如何在代码中正确处理中文？
**A**: 确保所有文件操作都使用UTF-8编码：
```pascal
// 读取文件
JsonText := TFile.ReadAllText(FileName, TEncoding.UTF8);

// 写入文件  
TFile.WriteAllText(FileName, JsonText, TEncoding.UTF8);
```
